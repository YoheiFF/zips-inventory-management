## 重要事項・既知の課題サマリー

### セキュリティ（本番稼働前に対応必須）

| 優先度 | 課題 | 参照 |
|-------|------|------|
| 高 | パスワード平文保存・平文比較 → bcrypt ハッシュ化が必要 | 04_detailed-design.md §5.1 |
| 中 | セッションメモリストア → Redis 等の外部ストア移行が必要 | 04_detailed-design.md §5.2 |
| 中 | Cookie secure=false → 本番環境では true に変更 | 04_detailed-design.md §5.2 |
| 中 | ロール制御未実装 → role カラムはあるが認可制御なし | 04_detailed-design.md §5.2 |

### DB 設計上の注意点

| 注意点 | 詳細 |
|-------|------|
| カラム名の命名規則の問題 | `AL_M_CATEGORY.item_name` と `AL_M_COMP.item_name` は実際には「カテゴリ名」「会社名」を表す。コーディング時に混乱しやすい |
| 在庫履歴なし | `AL_T_STOCK` は現在残数のみ保持。在庫変動の追跡不可 |
| 発注ヘッダーなし | `AL_T_ORDER` にヘッダーテーブルなし。`order_date` の GROUP BY で1発注をまとめている |

### 業務フロー上のリスク

| リスク | 説明 |
|-------|------|
| PDF とDB登録の不整合 | PDF 生成と発注DB登録が別 API 呼出のため、一方失敗時に不整合発生 |
| カートのブラウザ依存 | sessionStorage 使用のため、タブを閉じるとカートが消失 |
